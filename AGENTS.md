# Инструкции для AI-агентов

Этот репозиторий используется домовым комитетом для ведения списка
жильцов, протоколов собраний, решений и документов.

`AGENTS.md` — главный источник инструкций для AI-агентов в этом
репозитории.

## Язык

- Общаться с пользователем на русском языке, если он не попросил иначе.
- Сохранять имена жильцов и названия на иврите без транслитерации.
- Даты записывать в формате `YYYY-MM-DD`.

## Конфиденциальность

- Считать имена, телефоны и документы персональными или чувствительными
  данными.
- Не переносить содержимое репозитория во внешние сервисы без явного
  разрешения пользователя.
- Не публиковать и не цитировать телефонные номера без необходимости.
- Не создавать лишние копии файлов с персональными данными.

## Источники данных

- `residents.xlsx` — главный и актуальный список жильцов и телефонов.
- `residents.md` — только описание правил ведения списка; не дублировать
  в нём персональные данные.
- `meetings/` — протоколы собраний.
- `decisions/decisions.md` — краткий журнал окончательных решений.
- `documents/` — документы домового комитета.

Если данные противоречат друг другу, уточнить у пользователя, какой
вариант считать правильным. Не исправлять персональные данные по догадке.

## Правила изменений

- Сохранять существующую структуру и стиль файлов.
- Не удалять и не переименовывать файлы без необходимости.
- Не перезаписывать пользовательские изменения, не относящиеся к задаче.
- Для нового протокола копировать `meetings/template.md` и использовать
  имя `meetings/YYYY-MM-DD.md`.
- После собрания переносить окончательные решения в
  `decisions/decisions.md`.
- Документы именовать как `documents/YYYY-MM-DD-краткое-название.ext`.
- После изменения списка жильцов обновлять дату внутри `residents.xlsx`.

## Работа с Excel

- Хранить телефоны как текст в международном формате `+972...`.
- Не допускать преобразования телефонов в числа или экспоненциальную
  запись.
- Сохранять фильтры, закреплённую шапку и читаемое RTL-оформление.
- Перед завершением проверять, что имена и телефоны отображаются полностью.

## Git

- Не выполнять commit, push, merge или создание pull request без прямой
  просьбы пользователя.
- Перед завершением проверять изменённые файлы и отсутствие случайных
  временных файлов.
- Не добавлять секреты, токены, пароли и ключи доступа.

## Критерии готовности

- Изменения соответствуют запросу пользователя.
- Персональные данные не были случайно продублированы или раскрыты.
- Markdown-файлы читаемы, ссылки и пути корректны.
- Изменённые таблицы открываются и визуально проверены.
- Пользователю кратко сообщено, что именно изменено.
